Integrer Java og scripting med Spring
Leif J, 27. mars 2006 09:34
Craig Walls har i sin blogg skrevet en artikkel på hvordan man meget enkelt kan integrere java-kode og scripting-kode ved hjelp av Spring 2.0. Spring 2.0 introduserer klassene JRubyScriptFactory,GroovyScriptFactory og BshScriptFactory, som gjør det mulig å benytte hhv JRuby, Groovy og BeanShell-klasser som spring-injiserte bønner. I eksempelet Craig har laget benytter han JRuby til å definere en klasse i Ruby, for så å kalle dnene fra java via en ordinær bønne-definisjon i spring-configen. Ved hjelp av støtteklassen ScriptFactoryPostProcessor kan man konfigurere hvor ofte denne skal sjekke om script-filen har endret seg, og om nødvendig laste denne inn på nytt.

Er din JBoss appserver treg?
Leif J, 22. mars 2006 10:10
Andy C Oliver har på JBoss sin gruppeblogg ("Enter the JBoss Matrix") skrevet om noen enkle grep man kan gjøre for å få JBoss til å gå fortere. I standardkonfigurasjonen er JBoss appserveren satt opp til å benytte RMI, og RMI-subsystemet medfører full søppeltømming (garbage collection) hvert minutt. Dette betyr at alle tråder blir midlertidig avbrutt og minneområdet sveipet for objekter som kan kastes. På hardt belastede servere er dette en meget kostbar operasjon som med fordel kan gjøres sjeldnere. Andy forklarer hvordan, og mer til.

Mars 2006 (Bergen) GUI kveld med Swing og Qt

Tid: Torsdag 16. mars 2006, kl 19.00-21.30

Mars møtet blir viet til GUI.

Først ute er Yngvar Sørensen fra Bouvet. Yngaver var hos oss å snakket om AOP (Aspect Oriented Programming) og vil nå snakke om Swing. Vi har satset på avansert Swing og noen av godbitene er JTable capabilities and new JTable Mustang features. Threading strategies and alternatives. Swing worker. Look and Feel.

Det andre foredraget vil bli holdt av Jacek Suraszki fra Trolltech. Han vil gi oss en demo på utvikling med GUI bibliotekene Qt. Han har fått med seg Eskil Blomfeldt, som vil demonstrere Java-integrasjon for Qt. Java-integrasjonen er ikke ferdig, men er god nok for demo. Trolltech ønsker input fra oss på hva og hvordan de skal gå videre med i denne integrasjonen.
Både Jacek og Eskil jobber på Trolltech som utviklere og jobber med Qt bibliotekene og designeren.

Møte blir som vanlig på NITH og det blir javaPils i kroa etterpå.
